Mac Terminal 的使用

Posted on 2018-08-28 | Post modified: 2018-08-29 | In 技术 |

刚从 Windows 转到 Mac 不到两周，就发现 Mac 对开发者非常地友好。作为一个多年的 windows 使用者，刚接触 mac 的时候是一脸懵逼的，一时间还在傻乎乎地寻找 C 盘在哪 D 盘在哪，后来才发现 finder 很好用，非常喜欢 finder 的 tag 和 smart folder （推荐阅读一篇文章带你玩转Mac Finder）。Mac 的触控板的各种手势也是相当方便的，即使不用外置的鼠标也可以非常便捷地完成各种各样的操作。摸索完mac基本操作后，尝试把我之前在 windows 部署的 hexo blog 转移到 mac 中(Hexo GitHub pages Windows to Mac)，这过程中需要大量用到terminal的命令，借此机会写一篇 mac terminal 的使用，本文为 mac 新手使用 terminal 的分享，如有错漏，还请各位 mac 大神慷慨指正。

Command String的四个组成部分

  • Command name
  • Command options
  • Arguments
  • Extras
    其中，一条命令行中 command name 是必须的，后面三个（command options, arguments, extras)有时是可选的。Options 部分用-作为前导符号，其中许多命令的 Options 部分只包含单个字母，这时可以合并。如ls -lA与ls -l -A等价。Arguments 用来细化这个命令或指定这个命令具体的实施对象，Extras 则用来进一步实现其他功能

Terminal常用命令行/基本操作

  1. pwd print working directory，用于打印当前工作路径的绝对路径
  2. ls list directory contents，显示当前路径下的所有内容，包括文件和文件夹
  3. cd change directory，改变当前路径到指定路径（如果不指定则返回home folder），如：cd /Users/taylorliang/taylor/blog
  4. .. 上一级路径(parent directory)，举个例子，如果当前路径为/Users/taylorliang/taylor/blog，执行cd ..后当前路径会变成/Users/taylorliang/taylor
  5. ~ home folder，举个例子，如果当前路径为/Users/taylorliang/taylor/blog，执行cd ～后当前路径会变成/Users/taylorliang
  6. mkdir+文件夹名字 make directory, 新建文件夹
  7. rmdir+文件夹名字 remove directory, 有新建文件夹命令，那么当然就会有删除文件夹命令了，但是需要注意的是这个命令会永久删除文件夹，因为 command line 里面是没有废纸篓的，然而，rmdir 命令无法删除含有任何其他文件的文件夹，所以大多数情形下rmdir命令是不适用的。不过，可以利用rm添加-R选项来删除文件夹及包含的所有文件，请慎用。在此引用一下Renfei Song’s Blog关于command line的一句话：
    如果你输入了一些错误的命令，系统会返回一些错误信息。但是系统却不会阻止你做傻事（例如删除整个用户文件夹）。
  8. clear 用于清屏，也可以通过键盘组合control + L来完成clear屏幕的操作
  9. top 动态查看所有进程运行状况，按Q键退出

关于man的使用

Mac OS的man有点类似与 windows 的help，用于查看某个 command name 的使用方法。个人认为man跟pwd(print working directory)一样，应该是某个单词或者短语的缩写，我暂时没找到相关的说明，但是我猜应该是”manual”手册、小册子的意思。命令实在是太多了，完全记住根本是不可能的，因此需要有使用手册，man+command name可查看命令的使用说明，例如你想知道ls这个命令怎么使用，输入man ls即可进入使用说明页面。说明书/手册往往会很长，可以使用键盘的上下箭头进行浏览阅读，使用space（空格键）进行翻页操作，输入/和关键字来按照关键字搜索，按Q键退出说明书的阅读。如果连命令名称都不知道的话，还有一种万能的方法，输入man -k和关键字来对整个使用指南数据库进行搜索，再不行就只能google啦。

关于sudo

sudo，第一眼看还以为是“数独”的意思（读音很像嘛），然而它的意思是“substitute user do”，或者是“super user do”，用sudo执行一个命令会使用 root 账户权限。当然，使用之前需要 administrator 账户（管理员账户）的授权（如输入密码）。在上一篇博文Hexo GitHub pages Windows to Mac中，使用 npm 安装 hexo 时就使用到了sudo命令行，sudo npm install -g hexo。目前来说我对sudo仍了解不深，因此本版块暂时空着，等我学多一点再来写。

Terminal在Hexo博客搭建时

不得不说，Mac 对开发者非常友好。在 Windows 上使用 Git 前，必须要先安装 Git bash，然后通过 Git bash 来执行 git 命令行。而 Mac 自带 Git，无需安装 Git，因此此前在 windows 的 git bash 输入的命令行，直接在 terminal 上输入即可。

Terminal的其他使用

查看隐藏文件

为了简化工作，command line 和 Finder 都会隐藏许多文件和文件夹，这些内容通常是系统需要的。不借助第三方工具让 Finder 显示隐藏文件比较困难，但是在 command line 中却非常简单。首先，许多隐藏文件的隐藏是通过隐藏属性在 Finder 中隐藏的，而 command line 会忽略这些属性，所以这些文件会在 command line 中显示。另外，ls 命令会隐藏文件名以.开头的文件，但是这些文件却可以被显示出来，方法是利用-a选项。例如ls -la。
